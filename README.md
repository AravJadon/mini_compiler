# Mini Compiler for Toy-C Language

A complete 6-phase mini compiler implemented using **Lex** (Flex) and **Yacc** (Bison) that compiles a subset of C (Toy-C) down to RISC assembly. Each compilation phase produces distinct, observable output demonstrating the step-by-step transformation from source code to target code.

## 🔧 Build & Run

### Prerequisites
- **Flex** (lex)
- **Bison** (yacc)
- **GCC**
- **WSL** or Linux environment

### Build
```bash
make clean && make
```

### Run
```bash
./compiler < test_simple.c              # Run on simple test
./compiler < test_case_all.c            # Run on full test suite
./compiler < test_opt.c > output.txt    # Save output to file
```

### Manual Build (without Make)
```bash
bison -d parser.y
lex lexer.l
gcc -w -o compiler main.c symtab.c tac.c optimizer.c codegen.c lex.yy.c parser.tab.c
```

## 🏗️ Compilation Phases

### Phase 1 — Lexical Analysis (`lexer.l`)
Tokenizes the source code into a stream of tokens. Each token is logged with its line number, type, and lexeme.

**Output:**
```
  Line     Token Type           Lexeme
  ----     ----------           ------
  1        TYPE                 int
  1        ID                   main
  1        LPAREN               (
  ...
```

**Supported Tokens:** `TYPE`, `ID`, `NUMBER`, `STRING`, `IF`, `ELSE`, `FOR`, `WHILE`, `DO`, `SWITCH`, `CASE`, `DEFAULT`, `BREAK`, `CONTINUE`, `RETURN`, `PRINTF`, `SCANF`, all arithmetic/relational/logical operators, compound assignments (`+=`, `-=`, etc.), increment/decrement (`++`, `--`).

---

### Phase 2 — Syntax Analysis (`parser.y`)
Parses the token stream against the Toy-C grammar using an LALR(1) parser generated by Bison. Validates program structure and reports syntax errors with recovery.

**Grammar Supports:**
- Function definitions and prototypes
- Variable declarations with initializers
- Arithmetic expressions with operator precedence
- Control flow: `if/else`, `while`, `do-while`, `for`, `switch/case`
- I/O: `printf`, `scanf`
- Increment/decrement operators
- Compound assignment operators

---

### Phase 3 — Semantic Analysis (`symtab.c`)
Performs semantic checks using a hash-table-based symbol table:
- **Duplicate declaration detection** — flags variables declared more than once
- **Undeclared variable detection** — flags variables used before declaration
- **Symbol table dump** — lists all declared variables

---

### Phase 4 — Intermediate Code Generation (`tac.c`)
Generates Three-Address Code (TAC) from the parse tree using syntax-directed translation with backpatching for control flow.

**TAC Features:**
- Arithmetic expression decomposition into binary operations
- Temporary variable generation (`t1`, `t2`, ...)
- Short-circuit boolean evaluation
- Backpatching for `if/else`, `while`, `do-while` loops
- Function call representation (`param`, `call`)

**Example:**
```
  100: t1 = 3 + 5
  101: a = t1
  102: b = 10
  103: t2 = b + 2
  104: c = t2
```

---

### Phase 5 — Optimization (`optimizer.c`)
Applies four optimization passes on the TAC to reduce instruction count and improve efficiency:

| Pass | Description |
|------|-------------|
| **Constant Folding** | Evaluates constant expressions at compile time (`3 + 5` → `8`) |
| **Constant Propagation** | Substitutes known constant values into subsequent uses |
| **Common Sub-Expression Elimination (CSE)** | Reuses previously computed expressions instead of recomputing |
| **Dead Code Elimination** | Removes assignments to temporaries that are never used |

**Example Optimization:**
```
BEFORE:  t1 = 3 + 5       →   AFTER:  a = 8
         a = t1                        b = 10
         b = 10                        t2 = 12
         t2 = b + 2                    c = t2
         c = t2
```

---

### Phase 6 — Target Code Generation (`codegen.c`)
Translates optimized TAC into Simple RISC assembly (MIPS-like ISA).

**Target ISA:**

| Instruction | Description |
|---|---|
| `LOADI Rd, imm` | Load immediate value into register |
| `LOAD Rd, var` | Load variable from memory to register |
| `STORE Rs, var` | Store register to memory |
| `ADD Rd, Rs1, Rs2` | Addition |
| `SUB Rd, Rs1, Rs2` | Subtraction |
| `MUL Rd, Rs1, Rs2` | Multiplication |
| `DIV Rd, Rs1, Rs2` | Division |
| `MOD Rd, Rs1, Rs2` | Modulo |
| `CMP Rs1, Rs2` | Compare and set flags |
| `BEQ / BNE / BLT / BGT / BLE / BGE` | Conditional branches |
| `JMP label` | Unconditional jump |
| `HALT` | Stop execution |

**Example Output:**
```asm
.DATA
    a                    .WORD 0
    b                    .WORD 0

.CODE
    JMP L100
    ; a = 8
    LOADI R1, 8
    STORE R1, a
    ; t5 = d + c
    LOAD  R1, d
    LOAD  R2, c
    ADD   R1, R1, R2
    STORE R1, t5
    HALT
```

## 🧪 Supported Language Constructs (Toy-C)

```c
int main() {
    // Declarations
    int a, b = 10, c;

    // Arithmetic + compound assignments
    a = b + 5 * 2;
    a += 3;  a -= 1;  a *= 2;  a /= 3;  a %= 4;

    // Increment / Decrement
    a++;  ++b;  c--;  --a;

    // Control flow
    if (a > b && c <= 10 || !(a == 0)) { ... } else { ... }
    while (a < 100) { ... }
    do { ... } while (a > 0);
    for (int i = 0; i < 5; i > 0) { ... }
    switch (a) { case 1: { ... } case 2: { ... } default: { ... } }

    // I/O
    printf("Hello");
    printf("Value: ", a);
    scanf("%d", &b);

    // Return
    return 0;
}
```
