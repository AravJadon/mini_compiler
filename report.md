# Mini Compiler for Toy-C Language

A complete 6-phase mini compiler implemented using **Lex (Flex)** and **Yacc (Bison)** that compiles a subset of C (Toy-C) down to a custom RISC assembly. 

This project is deeply rooted in core **Compiler Design** concepts. It demonstrates the complete translation pipeline, starting from raw character streams, continuing through rigorous semantic validation and advanced aggressive code optimization, and ultimately targeting a register-constrained architecture using graph coloring.

---

## What Compiler Concepts Are Used?

This compiler leverages standard, industry-recognized compilation techniques across its 6 distinct phases:

1. **Regular Expressions & Finite Automata** for Lexical Analysis.
2. **Context-Free Grammars & LALR(1) Parsing** for Syntax Analysis, handling classic shift/reduce ambiguities (like *dangling-else* via precedence rules).
3. **Syntax-Directed Translation (SDT)** for on-the-fly intermediate code generation.
4. **Symbol Tables** using hash maps to enforce scope and Semantic Analysis.
5. **Backpatching** to resolve forward jumps for complex control flows (`if-else`, nested `while`, `do-while`, `for`) in a single pass.
6. **Data-Flow Analysis & Loop Analysis** for comprehensive intermediate code optimization.
7. **Liveness Analysis & Graph-Coloring Register Allocation** (Chaitin's Algorithm concepts) to assign an infinite number of intermediate variables to a bounded hardware register file.


## The 6 Compilation Phases

### Phase 1 ‚Äî Lexical Analysis (`lexer.l`)
Tokenizes the source code into a stream of tokens. It uses **Regular Expressions** compiled into a Deterministic Finite Automaton (DFA) by Flex. It automatically filters out whitespace and comments while tracking line and column numbers for error reporting.

### Phase 2 ‚Äî Syntax Analysis (`parser.y`)
Parses the token stream against the Toy-C grammar using an LALR(1) parser generated by Bison. 
**Key Concepts**: 
- Validates structural correctness based on a **Context-Free Grammar**.
- Resolves classic grammar ambiguities (like the *dangling-else* problem) gracefully using Bison precedence rules (`%prec`). 
- Provides robust **Syntax Error Recovery**, preventing cascading parse failures by discarding tokens until a safe synchronization point (like a semicolon) is found.

### Phase 3 ‚Äî Semantic Analysis (`symtab.c`)
Performs contextual checks that the grammar alone cannot enforce.
**Key Concepts**:
- Uses an efficient **Hash-Table-based Symbol Table**.
- Detects **Duplicate Declarations** (e.g., throwing an error if a variable is declared twice in the same scope).
- Detects **Undeclared Usage** (e.g., using a variable before defining it).
- Tracks whether a variable has been initialized.

### Phase 4 ‚Äî Intermediate Code Generation (`tac.c`)
Converts the parsed grammar into platform-independent **Three-Address Code (TAC)**.
**Key Concepts**:
- **Syntax-Directed Translation**: Generates code during the parsing (reduction) phase.
- **Short-Circuit Evaluation**: Translates boolean logic (`&&`, `||`) directly into jumps without executing unnecessary conditions.
- **Backpatching**: Uses lists of unresolved jump targets (`truelist`, `falselist`, `nextlist`) to patch the destination addresses of `GOTO` statements dynamically once the destination block has been evaluated. 

### Phase 5 ‚Äî Optimization (`optimizer.c`)
The compiler utilizes multiple sophisticated optimization passes over the TAC. It runs repeatedly until a **fixed point** is reached (no further optimizations can be applied).

| Pass | Concept / Action |
|------|-------------|
| **Constant Folding** | Evaluates explicit numerical expressions at compile time (`3 + 5` ‚Üí `8`) |
| **Algebraic Simplification** | Reduces math identities (`x * 1` ‚Üí `x`, `y + 0` ‚Üí `y`) |
| **Strength Reduction** | Replaces expensive operations with cheaper ones (`x * 2` ‚Üí `x + x`) |
| **Constant Propagation (Local/Global)** | Replaces variable references with their known constant values across statements and blocks |
| **Copy Propagation** | Replaces uses of a variable with a directly assigned alias/temp to reduce indirection |
| **Common Sub-Expression Elimination (CSE)** | Detects if an identical calculation has already been done and reuses the result rather than recomputing it |
| **Constant-Condition Folding** | Evaluates boolean branch conditions at compile time (`if 100 < 0` ‚Üí `false`) |
| **Unreachable Code Elimination** | Identifies branches, jumps, or loops whose entry conditions are perpetually false, pruning them entirely |
| **Loop-Invariant Code Motion (LICM)** | Analyzes loops to find calculations whose inputs don't change inside the loop. The calculations are recursively hoisted natively outside of nested loops to execute only once. |
| **Dead Code Elimination** | Safely removes variable assignments whose values are subsequently never read |

### Phase 6 ‚Äî Target Code Generation (`codegen.c`)
Translates the highly-optimized TAC into a Simple RISC assembly (MIPS-like ISA).
**Key Concepts**:
- **Liveness Analysis** calculates which variables are "live" (needed later in execution) at each instruction step.
- **Interference Graphs** are constructed where edges represent variables that are live at the same time (and thus cannot share hardware registers).
- **Chaitin‚Äôs Graph Coloring Algorithm** allocates a theoretical infinite number of temporary bindings into exactly 8 physical hardware registers (`R0-R7`).

##  Supported Language Constructs (Toy-C)

```c
int main() {
    // Declarations
    int a, b = 10, c;

    // Arithmetic + compound assignments
    a = b + 5 * 2;
    a += 3;  a -= 1;  a *= 2;  a /= 3;  a %= 4;

    // Increment / Decrement
    a++;  ++b;  c--;  --a;

    // Control flow
    if (a > b && c <= 10 || !(a == 0)) { ... } else { ... }
    while (a < 100) { ... }
    do { ... } while (a > 0);
    for (int i = 0; i < 5; i > 0) { ... }
    switch (a) { case 1: { ... } case 2: { ... } default: { ... } }

    // I/O
    printf("Hello");
    printf("Value: ", a);
    scanf("%d", &b);

    // Return
    return 0;
}
```
